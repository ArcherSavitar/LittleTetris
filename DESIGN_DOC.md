# LittleTetris 项目设计文档（基于现有实现）

## 1. 项目概述
- 项目类型：纯前端单机游戏（HTML + CSS + JavaScript，无后端）
- 目标：实现经典俄罗斯方块核心玩法，并提供基础 UI（分数/等级/消行、下一个方块预览、开始/暂停/结束）
- 运行方式：浏览器直接打开 `index.html`

## 2. 当前功能范围
- 10x20 棋盘
- 7 种方块（I, J, L, O, S, T, Z）
- 键盘操作：左右移动、旋转、软降、硬降、暂停
- 方块碰撞检测、落地锁定、消行
- 计分与等级提升（随消行数提升）
- 下一个方块预览
- 按钮控制：开始、暂停/继续、结束
- 游戏结束判定与遮罩提示

## 3. 系统结构

### 3.1 文件结构
- `index.html`：页面结构与脚本加载顺序
- `css/style.css`：布局和基础视觉样式
- `js/utils/constants.js`：常量配置（棋盘尺寸、方块定义、颜色、等级速度）
- `js/core/Tetromino.js`：方块实体与移动/旋转逻辑
- `js/core/Board.js`：棋盘数据、合法性判断、消行、计分、等级
- `js/core/Game.js`：游戏主控制器与主循环
- `js/ui/UI.js`：Canvas 渲染与 UI 数字更新
- `js/main.js`：DOM 事件与按钮逻辑绑定

### 3.2 模块职责
- `Game`：统一管理游戏状态（是否开始、暂停、结束）、驱动帧循环、协调 `Board` 与 `UI`
- `Board`：维护二维网格与规则计算，是核心“规则层”
- `Tetromino`：维护当前方块形状与位置，是“实体层”
- `UI`：负责“表现层”，将状态绘制到主棋盘和预览面板

## 4. 核心数据模型
- 棋盘：`grid[height][width]`，`0` 表示空格，`1..7` 表示方块类型
- 当前方块：`type + shape + row + col`
- 游戏统计：`score`、`lines`、`level`、`clearedLines`
- 时序控制：`dropCounter`、`dropInterval`、`lastTime`

## 5. 核心流程设计

### 5.1 生命周期
1. 初始化 `Game`，创建当前方块与下一个方块
2. 用户点击“开始游戏”后进入 `requestAnimationFrame` 循环
3. 按 `dropInterval` 驱动自动下落
4. 若方块不能继续下落：锁定到棋盘 -> 消行与计分 -> 切换新方块
5. 新方块初始位置非法则游戏结束

### 5.2 输入控制
- 键盘事件在 `Game.bindEvents()` 统一处理
- 仅在“已开始且未结束”时响应输入
- 暂停状态下屏蔽方块移动/旋转/下落（仅允许取消暂停）

### 5.3 渲染策略
- 每帧调用 `UI.render()`
- 主棋盘 Canvas：先清空背景，再绘制固定块和当前块
- 预览 Canvas：独立绘制下一个方块并居中
- 覆盖层：暂停/结束时在主棋盘中央绘制提示

## 6. 规则实现说明
- 移动合法性：边界检查 + 已落地方块碰撞检查
- 旋转：矩阵转置 + 反序（顺时针），非法则回滚
- 消行：从底向上扫描，满行即删除并顶部补空行
- 计分：
  - 1/2/3/4 行分别为 100/300/500/800（再乘等级）
  - 硬降附加分：每格 `2 * level`
- 等级：每累计消除 10 行升 1 级，并影响下落速度

## 7. 状态机（现状）
- `isStarted=false`：未开始
- `isStarted=true && !isPaused && !gameOver`：进行中
- `isPaused=true`：暂停中
- `gameOver=true`：结束

按钮与键盘共同驱动状态切换，但“按钮文案状态”和“游戏状态”分散在 `main.js` 与 `Game.js`，后续可统一。

## 8. 已知问题与风险
- `Game.js` 内存在 `window.onload = () => new Game()`，`main.js` 内又在 `DOMContentLoaded` 中 `new Game()`，存在重复初始化风险（逻辑冲突、事件重复绑定）。
- 使用 `keyCode`（已过时），建议迁移到 `event.key`。
- 旋转未实现 wall kick（贴墙旋转补偿），边界手感较弱。
- 随机方块为简单均匀随机，未使用 7-bag，可能出现分布极端情况。
- UI 与游戏状态耦合偏高，缺少统一状态通知机制。

## 9. 非功能设计
- 性能：当前 10x20 网格和 60fps 目标下开销很小，浏览器可稳定运行
- 可维护性：模块已拆分为 `core/ui/utils`，但依赖为全局变量方式，缺少模块化构建体系
- 兼容性：现代浏览器可运行；键盘事件行为在不同系统可能有细微差异

## 10. 后续演进建议（按优先级）
1. 先修复双初始化问题（保留一个入口初始化点）
2. 引入 7-bag 随机算法，提升公平性
3. 增加 wall kick 旋转规则，优化操作手感
4. 增加“锁定延迟（lock delay）”“连击/B2B/T-Spin”等进阶规则
5. 抽离状态管理与事件总线，降低 `Game/UI/main` 间耦合
6. 增加自动化测试（规则层优先：碰撞、消行、计分、升级）
7. 逐步迁移到 ES Module 或 TypeScript，提升工程化与可扩展性

## 11. 验收基线（建议）
- 功能：键盘与按钮控制均可用，得分/等级/消行正确更新
- 稳定性：连续运行 10 分钟无明显卡顿/异常
- 正确性：核心规则（碰撞、消行、结束）覆盖用例通过
- 体验：暂停/继续/结束状态切换一致、无 UI 文案错乱
